# Excel 自動同期システム ― 仕様・機能・注意事項

NAS上のExcelファイルを Google スプレッドシートへ自動で反映するシステムの仕様書です。

---

## 1. システムの概要

本システムは、社内NASに保存されているExcelファイルの内容を、一定間隔で自動的にGoogleスプレッドシートへ反映します。

これにより、**事務員の方は今まで通りExcelで作業**しながら、**営業・役員・社長の方はスマートフォンやPCから外出先でスプレッドシートを閲覧**できるようになります。

```
┌──────────┐          ┌──────────┐           ┌──────────────────┐
│ NAS      │  自動で  │ 社内PC   │  自動で   │ Google           │
│ (Excel)  │ ───────► │ (同期    │ ────────► │ スプレッドシート  │
│          │  読み取り │  システム) │  書き込み  │                  │
└──────────┘          └──────────┘           └──────────────────┘
  事務員が                15分ごとに              営業・役員が
  いつも通り編集          自動で処理              外出先から閲覧
```

---

## 2. できること

| 機能 | 説明 |
|------|------|
| 自動同期 | Excelファイルの更新を検知し、スプレッドシートへ自動反映します |
| 複数ファイル対応 | NAS内のExcelファイル（10〜20個程度）をまとめて管理します |
| 複数シート対応 | 1つのExcelファイル内にシートが複数あっても、すべて反映されます |
| サブフォルダ対応 | NASフォルダ内にサブフォルダがある場合も、中のExcelを検出します |
| 新規ファイル自動検知 | NASに新しいExcelを追加すると、自動でスプレッドシートが作成されます |
| URL固定 | 一度作成されたスプレッドシートのURLは変わりません。ブックマーク可能です |
| 閲覧者の自動設定 | 指定したメールアドレスに対し、閲覧権限が自動で付与されます |
| 常時自動運用 | 社内PCで自動起動し、PC再起動後も自動で復帰します |

---

## 3. 同期の仕組み

### 3-1. 同期の方向

**NASのExcel → スプレッドシート の一方向のみ**です。

- 事務員がExcelを更新 → スプレッドシートに反映される
- スプレッドシート上で編集しても → Excelには反映されません

スプレッドシートは**閲覧専用**としてご利用ください。

### 3-2. 同期のタイミング

**15分に1回**、自動で同期処理が実行されます。

```
[例]
10:00  同期チェック → 変更あり → 反映
10:15  同期チェック → 変更なし → 何もしない
10:30  同期チェック → 変更あり → 反映
  :
```

- 事務員がExcelを保存してから、**最大15分後**にスプレッドシートへ反映されます
- 変更がないファイルは処理をスキップするため、無駄な通信は発生しません
- 同期間隔は設定で変更可能です（例: 5分、30分 など）

### 3-3. 変更の検知方法

Excelファイルの**更新日時**と**ファイル内容**の両方を確認して変更を検知します。内容に変化がなければ、保存しただけでは同期は実行されません。

## 4. 閲覧中に更新が入った場合の動作

**スプレッドシートが一瞬空白になることはありません。**

スプレッドシートを閲覧中にちょうど更新タイミングが来た場合、以下のように動作します。

| タイミング | 画面に表示される内容 |
|-----------|-------------------|
| 更新処理の前 | 前回同期時のデータ（古いデータ） |
| 更新処理の実行中 | 古いデータが新しいデータに直接切り替わる |
| 更新処理の後 | 最新データ |

既存のデータを消してから新しいデータを書き込むのではなく、**上から直接新しいデータで上書き**します。そのため、閲覧中に画面が真っ白になることはありません。

> ただし、更新の切り替わる瞬間は一部のセルだけが新しい値に変わるなど、一時的に新旧データが混在して見える場合があります。数秒で完了しますので、ページを再読み込みすれば最新データが表示されます。

## 5. ファイルの追加・削除・名前変更

### 新しいExcelファイルをNASに追加した場合

次の同期タイミングで自動検知し、新しいスプレッドシートが自動作成されます。
設定済みの閲覧者への共有も自動で行われます。

> 新しいスプレッドシートのURLは、Google Drive の同期用フォルダ内に作成されます。
> URLを営業・役員へ共有してください。

### NASからExcelファイルを削除した場合

**対応するスプレッドシートは削除されません。**
最後に同期されたデータがそのまま残ります。これはデータ保全のための仕様です。

不要になったスプレッドシートは、Google Drive上で手動で削除してください。

### Excelファイルの名前を変更した場合

- 旧ファイル名のスプレッドシートはそのまま残ります（削除と同じ扱い）
- 新ファイル名で新しいスプレッドシートが作成されます
- 営業・役員には**新しいURLをお知らせ**する必要があります

> ファイル名の変更は実質的に「旧ファイル削除 + 新ファイル追加」として扱われます。
> 頻繁なファイル名の変更は避けることをお勧めします。

### Excelのシート構成を変更した場合

| 操作 | 動作 |
|------|------|
| シートの追加 | 次回同期時にスプレッドシート側にもシートが追加されます |
| シートの削除 | 次回同期時にスプレッドシート側のシートも削除されます |
| シート名の変更 | 旧シートが削除され、新しい名前のシートが作成されます |

## 6. 事務員の方への影響

**ありません。** 今まで通りの作業で問題ありません。

- NAS上のExcelをいつも通り開いて、編集して、保存してください
- 特別な操作や手順の変更は一切不要です
- Excelの動作が遅くなることもありません

事務員がExcelを編集中（開いている状態）でも、システムは正常に動作します。
Excelが保存されるたびに、次の同期タイミングで反映されます。

## 7. 日付・数値の表示について

Excelのデータは、**入力された値がそのまま**スプレッドシートに転記されます。

| Excelでの入力 | スプレッドシートでの表示 |
|--------------|----------------------|
| 数値（123, 45.6） | そのまま表示 |
| 文字列（在庫あり） | そのまま表示 |
| 日付（2026/3/17） | 2026-03-17 形式で表示 |
| 日時（2026/3/17 10:30） | 2026-03-17 10:30:00 形式で表示 |
| 数式（=SUM(A1:A10)） | 計算結果の値が転記されます（数式自体は転記されません） |
| 空セル | 空白のまま |

### 注意点

- **Excelの数式**は「計算結果の値」のみが反映されます。スプレッドシート側に数式は入りません
- **セルの書式**（色、太字、罫線など）は反映されません。データの値のみが対象です
- **グラフ・画像**は反映されません
- **日付の表示形式**は Excel とスプレッドシートで異なる場合があります（データの値自体は正確です）

## 8. スプレッドシートの閲覧について

### アクセス方法

Google Drive 内の指定フォルダにスプレッドシートが格納されます。
各スプレッドシートの名前は `[同期] Excelファイル名` という形式です。

例:
- `[同期] 在庫表`
- `[同期] 売上管理`
- `[同期] 顧客リスト`

### ブックマーク

スプレッドシートのURLは固定です。
一度ブラウザでブックマークしておけば、次回以降は同じURLで最新データを確認できます。

### スマートフォンからの閲覧

Google スプレッドシートアプリ（iOS / Android）から閲覧できます。
Google アカウントでログインし、共有されたスプレッドシートを開いてください。

### 閲覧権限

- 営業・役員の方には**閲覧のみ**の権限が付与されます
- スプレッドシート上での編集はできません（誤操作防止のため）
- 閲覧者の追加・削除は管理者が設定変更で対応します

## 9. システムが正常に動作しない場合

以下のような状況では、同期が一時的に停止または遅延する場合があります。

| 状況 | 影響 | 自動復旧 |
|------|------|---------|
| NASの電源が切れている | 同期が停止（スプレッドシートは最後のデータを保持） | NAS復旧後、自動で再開 |
| 社内ネットワークの障害 | 同期が停止 | ネットワーク復旧後、自動で再開 |
| インターネット接続の障害 | Google への送信ができない | 接続復旧後、自動で再開 |
| 同期用PCの電源が切れた | 同期が停止 | PC再起動後、自動で再開 |
| Excelファイルが破損している | 該当ファイルのみスキップ、他のファイルは正常に同期 | ファイル修復後、自動で再開 |

いずれの場合も、**スプレッドシート上のデータが消えることはありません。**
最後に同期された時点のデータが保持されます。

## 10. 現時点での制約事項

| 項目 | 内容 |
|------|------|
| 同期の方向 | NAS → スプレッドシートの一方向のみ。逆方向はできません |
| リアルタイム性 | 最大15分の遅延があります（即時反映ではありません） |
| 書式・装飾 | セルの色、フォント、太字、罫線などは反映されません |
| グラフ・画像 | Excel内のグラフや画像は反映されません |
| マクロ（VBA） | マクロの内容は反映されません（マクロの実行結果は値として反映されます） |
| .xls形式 | 旧形式（.xls）のExcelファイルには対応していません。.xlsx形式のみ対象です |
| 同時編集 | 複数名がスプレッドシートで同時編集する用途には対応していません（閲覧専用） |
| 通知機能 | 同期の成功・失敗を自動通知する機能は現時点では未搭載です |

## 11. よくあるご質問

**Q. 事務員がExcelを開いたまま（編集中）でも同期されますか？**
A. はい。Excelが保存された時点の内容が、次の同期タイミングで反映されます。ただし、保存せずに開いたままの編集中の内容は反映されません。

**Q. 同期間隔を変更できますか？**
A. はい。設定ファイルで変更可能です（例: 5分、10分、30分など）。変更後はサービスの再起動が必要です。

**Q. スプレッドシートを誤って削除してしまった場合は？**
A. NAS上のExcelファイルが残っていれば、次の同期タイミングで新しいスプレッドシートが自動作成されます。ただしURLが変わりますので、ブックマークの更新が必要です。

**Q. 閲覧者を追加・削除したい場合は？**
A. 管理者にご依頼ください。設定ファイルの変更で対応いたします。また、Google Drive上で手動での共有追加も可能です。

**Q. Excelの数式の計算結果が反映されていません**
A. Excelを開いて一度再計算（Ctrl + Shift + F9）してから保存してください。Excelが最後に計算した結果の値が同期されます。

**Q. NASのフォルダ構成を変更しても大丈夫ですか？**
A. 同期対象フォルダの配下であれば、サブフォルダの追加・移動は自動対応されます。ただし、同期対象フォルダ自体のパスを変更する場合は、設定の変更が必要です。

**Q. 同期が止まっているかもしれません。確認方法は？**
A. スプレッドシートの内容がしばらく更新されていない場合は、管理者にご連絡ください。同期用PCのログを確認して対応いたします。
