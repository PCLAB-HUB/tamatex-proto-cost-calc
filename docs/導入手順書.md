## 1. 前提条件

以下が揃っていることを確認してください。

| 項目 | 要件 |
|------|------|
| 常時稼働PC | Windows 10/11、ネットワーク接続あり |
| Python | バージョン 3.11 以上 |
| QNAP NAS | PCからネットワークドライブとしてアクセス可能 |
| Google Workspace | 管理者権限を持つアカウント |
| インターネット | 常時接続（Google API通信用） |

### Python のインストール確認

コマンドプロンプトを開き、以下を実行します。

```
python --version
```

`Python 3.11.x` 以上が表示されればOKです。
表示されない場合は https://www.python.org/downloads/ からインストールしてください。

> **重要**: インストール時に **「Add Python to PATH」にチェック** を入れてください。

### NAS接続の確認

エクスプローラーでNASのExcelファイルが保存されているフォルダにアクセスできることを確認します。

**ドライブレターでアクセスしている場合（推奨）:**
```
Z:\共有\Excel
```

**UNCパスでアクセスしている場合:**
```
\\192.168.1.100\shared\Excel
```

> **推奨**: UNCパスよりドライブレターの方が安定します。ネットワークドライブの割り当て手順は[付録A](#付録a-ネットワークドライブの割り当て)を参照してください。

## 2. 導入方法の選択

tamatexには2つの導入方法があります。

| 方法 | 対象 | 所要時間 | 概要 |
|------|------|----------|------|
| **A. GUIインストーラー（推奨）** | 全ユーザー | 約10〜15分 | 7ステップのウィザードに従うだけで完了 |
| **B. 手動インストール** | IT技術者向け | 約20〜30分 | コマンドラインで各手順を実行 |

**はじめての方は方法Aを推奨します。** GUIインストーラーはGoogle Cloud設定からサービス起動まで、すべてのステップをガイド付きで案内します。

---

## 3. 方法A: GUIインストーラーで導入（推奨）

GUIインストーラーウィザードが、以下のすべてを案内・自動実行します。

- Google Cloud プロジェクトとAPI設定のガイダンス
- サービスアカウント作成のガイダンスとキーファイル選択
- Google Driveフォルダ設定のガイダンス
- NASパス・同期間隔などの構成入力
- ファイルコピー、仮想環境作成、パッケージインストールの自動実行
- 設定ファイル（config.yaml）の自動生成
- Windowsサービスとしての登録・起動

### 3-1. インストーラーの起動

tamatexフォルダ内の `セットアップ.bat` をダブルクリックして起動します。

```
tamatex\セットアップ.bat
```

> **exe版がある場合**: `Excel同期セットアップ.exe` をダブルクリックしてください（Pythonのインストールが別途必要です）。

ウィザード画面が表示されます。左側のサイドバーに全7ステップが表示され、現在のステップがハイライトされます。

### 3-2. ステップ1: ようこそ

ウィザードの概要が表示されます。

- **所要時間**: 約10〜15分（Google Cloud設定を含む）
- **できること**: NAS上のExcelファイル自動検出、スプレッドシートへの定期同期

内容を確認し、**「次へ」** をクリックします。

### 3-3. ステップ2: Google Cloud設定

Google Sheets APIを利用するために、Google Cloudプロジェクトを準備します。ウィザードの指示に従って、ブラウザ上で以下を実施してください。

**操作手順:**

1. ウィザード画面の **「Google Cloud Consoleを開く」** ボタンをクリック
2. Google Workspaceの管理者アカウントでログイン
3. 画面上部のプロジェクト選択ドロップダウンから **「新しいプロジェクト」** をクリック
4. プロジェクト名に `tamatex` と入力して **「作成」** をクリック
5. 作成したプロジェクトが選択された状態で、左メニューの **「APIとサービス」→「ライブラリ」** を開く
6. 以下の2つのAPIを有効化:
   - 検索バーで `Google Sheets API` を検索 → **「有効にする」** をクリック
   - 検索バーで `Google Drive API` を検索 → **「有効にする」** をクリック

**完了後**: ウィザード画面の **「APIの有効化が完了しました」** にチェックを入れ、**「次へ」** をクリックします。

### 3-4. ステップ3: サービスアカウント

サービスアカウントは、tamatexがGoogle APIにアクセスするための専用アカウントです。

**操作手順:**

1. ウィザード画面の **「認証情報ページを開く」** ボタンをクリック
2. ブラウザで認証情報ページが開く（先ほど作成したプロジェクトが選択されていることを確認）
3. **「+ サービスアカウントを作成」** をクリック
4. 以下を入力:
   - サービスアカウント名: `tamatex-sync`（わかりやすい任意の名前）
   - サービスアカウントID: 自動入力される
   - 説明: `Excel to Spreadsheet sync`（任意）
5. **「作成して続行」** をクリック
6. ロールの選択は **スキップ**（「続行」→「完了」をクリック）
7. 作成されたサービスアカウント `tamatex-sync` をクリック
8. **「キー」タブ** を開く
9. **「鍵を追加」→「新しい鍵を作成」** をクリック
10. キーのタイプ: **JSON** を選択 →「作成」
11. JSONファイルが自動的にダウンロードされる

> **重要**: このJSONファイルは認証情報です。第三者に渡さないでください。

**ウィザードに戻って:**

12. **「参照...」** ボタンをクリックし、ダウンロードしたJSONファイルを選択
13. サービスアカウントのメールアドレスが表示されたことを確認（例: `tamatex-sync@tamatex-xxxxx.iam.gserviceaccount.com`）
14. **「次へ」** をクリック

### 3-5. ステップ4: Google Drive設定

同期先のスプレッドシートを格納するGoogle Driveフォルダを準備します。

**操作手順:**

1. ウィザード画面の **「Google Driveを開く」** ボタンをクリック
2. 「マイドライブ」または「共有ドライブ」内に新しいフォルダを作成（例: `NAS同期データ`）
3. 作成したフォルダを開き、ブラウザのアドレスバーからフォルダIDをコピー

```
https://drive.google.com/drive/folders/1ABCdefGHIjklMNOpqrSTUvwxYZ
                                        ^^^^^^^^^^^^^^^^^^^^^^^^^^^^
                                        この部分がフォルダID
```

4. フォルダを右クリック →「共有」→「ユーザーやグループと共有」
5. ウィザード画面に表示されているサービスアカウントのメールアドレスをコピーして貼り付け
6. 権限を **「編集者」** に設定 →「送信」

> **重要**: この共有設定を行わないと、サービスアカウントがフォルダにファイルを作成できません。

**ウィザードに戻って:**

7. **「フォルダID」** 欄にコピーしたIDを貼り付け
8. **「フォルダの共有設定が完了しました」** にチェックを入れる
9. **「次へ」** をクリック

### 3-6. ステップ5: 構成入力

同期設定とインストール先を入力します。

| 入力項目 | 説明 | 入力例 |
|----------|------|--------|
| **NASパス** | NAS上のExcelフォルダのパス | `Z:\共有\Excel` |
| **インストール先** | tamatexをインストールするフォルダ | `C:\tamatex` |
| **同期間隔** | 同期の実行間隔（分） | `15`（デフォルト） |
| **共有メールアドレス** | スプレッドシートを共有する営業・役員のメールアドレス（1行1アドレス） | `eigyo@example.com` |

**NASパスの入力:**
- 「参照...」ボタンでフォルダ選択ダイアログから選択できます
- NASが接続されていない場合は、パスを手入力して「はい」で続行できます

**共有メールアドレス:**
- 1行に1つのメールアドレスを入力
- 空欄のままでもOK（後から `config.yaml` に追加可能）

入力が完了したら **「次へ」** をクリックします。

### 3-7. ステップ6: インストール

**「インストール開始」** ボタンをクリックすると、以下の処理が自動実行されます。

| ステップ | 処理内容 | 所要時間 |
|----------|----------|----------|
| [1/7] | インストールディレクトリ作成 | 数秒 |
| [2/7] | プロジェクトファイルのコピー | 数秒 |
| [3/7] | 設定ファイル（config.yaml）生成 | 数秒 |
| [4/7] | サービスアカウントキーのコピー | 数秒 |
| [5/7] | Python仮想環境の作成 | 約30秒 |
| [6/7] | tamatexパッケージのインストール | 約1〜3分 |
| [7/7] | ログディレクトリの確認 | 数秒 |

進捗バーとログが画面に表示されます。すべて緑色の「完了」表示になるまでお待ちください。

> **注意**: インストール中はウィザードを閉じないでください。

エラーが発生した場合は、ログ欄に赤色でエラー内容が表示されます。内容を確認し、問題を解消してから再試行してください。

### 3-8. ステップ7: 完了

インストールが成功すると、以下が表示されます。

- **インストール概要**: 設定内容のサマリー（インストール先、NASパス、同期間隔など）
- **同期サービスの開始ボタン**: クリックするとNSSMでWindowsサービスとして登録・起動します

**同期サービスの開始:**

1. **「同期サービスを開始」** ボタンをクリック
2. NSSMがインストールされている場合 → 自動的にサービス登録・起動
3. NSSMが見つからない場合 → 手動実行用のコマンドが表示されます

**NSSMが無い場合の手動起動:**

画面に表示されるコマンドをコマンドプロンプトにコピーして実行します。

```bat
cd C:\tamatex
C:\tamatex\.venv\Scripts\python.exe -m tamatex.main -c config\config.yaml
```

> NSSMのインストール方法は[7-5. NSSMのインストール](#7-5-nssmのインストール)を参照してください。

**「閉じる」** ボタンでウィザードを終了します。

**次のステップ**: [5. 動作確認](#5-動作確認) に進んでください。

## 4. 方法B: 手動で導入（上級者向け）

コマンドラインに慣れたIT技術者向けの手動導入手順です。

> **方法AのGUIインストーラーで導入済みの場合、この章はスキップしてください。**

### 4-1. Google Cloud プロジェクトの準備

[3-3. ステップ2: Google Cloud設定](#3-3-ステップ2-google-cloud設定)と同じ手順で、Google Cloud ConsoleからAPIを有効化します。

1. https://console.cloud.google.com/ にアクセス
2. 新しいプロジェクトを作成（名前: `tamatex`）
3. Google Sheets API と Google Drive API を有効化

### 4-2. サービスアカウントの作成

[3-4. ステップ3: サービスアカウント](#3-4-ステップ3-サービスアカウント)と同じ手順で作成します。

1. 「APIとサービス」→「認証情報」→「＋認証情報を作成」→「サービスアカウント」
2. 名前: `tamatex-sync` で作成
3. 「キー」タブ → JSON形式でキーをダウンロード
4. ダウンロードしたファイルの名前を `service_account.json` に変更

JSONファイルを開き、`client_email` の値を控えてください（次のステップで使用）。

```json
{
  "client_email": "tamatex-sync@tamatex-xxxxx.iam.gserviceaccount.com"
}
```

### 4-3. Google Drive フォルダの準備

1. https://drive.google.com/ でフォルダを作成（例: `NAS同期データ`）
2. URLからフォルダIDを取得
3. フォルダをサービスアカウントのメールアドレスと共有（「編集者」権限）

### 4-4. プロジェクトファイルの配置

tamatex フォルダを常時稼働PCの任意の場所にコピーします。

```
推奨配置例: C:\tamatex\
```

### 4-5. サービスアカウントキーの配置

ダウンロードした `service_account.json` を以下に配置します。

```
C:\tamatex\config\service_account.json
```

### 4-6. Python 仮想環境の作成とパッケージインストール

コマンドプロンプトを **管理者として実行** し、以下を順に実行します。

```bat
cd C:\tamatex

python -m venv .venv

.venv\Scripts\activate

pip install .
```

> **重要**: `pip install -r requirements.txt` ではなく **`pip install .`** を使用してください。`pip install .` は依存パッケージに加えてtamatexパッケージ本体もインストールし、`tamatex` コマンドが使えるようになります。

以下のようなメッセージが表示されれば成功です。

```
Successfully installed gspread-6.x.x google-auth-2.x.x openpyxl-3.x.x pyyaml-6.x.x tamatex-0.1.0
```

### 4-7. 設定ファイルの作成

```bat
cd C:\tamatex
copy config\config.example.yaml config\config.yaml
```

`config\config.yaml` をテキストエディタ（メモ帳等）で開き、以下を編集します。

```yaml
nas:
  # ★ NASのExcelフォルダパスを入力
  # Windowsパスの場合、バックスラッシュは二重にする
  base_path: "Z:\\共有\\Excel"
  file_patterns:
    - "*.xlsx"
  exclude_patterns:
    - "~$*"
    - "*.tmp"
    - ".~lock*"

google:
  # ★ サービスアカウントキーのパス（手順4-5で配置した場所）
  credentials_path: "./config/service_account.json"
  # ★ 手順4-3で控えたGoogle DriveフォルダID
  drive_folder_id: "1ABCdefGHIjklMNOpqrSTUvwxYZ"
  # ★ スプレッドシートを共有するメールアドレス
  share_with:
    - "eigyo@example.com"
    - "yakuin@example.com"

sync:
  # 同期間隔（分）
  interval_minutes: 15
  # 状態DBファイルのパス（空なら設定ファイルと同じディレクトリに自動作成）
  # state_db_path: "C:\\tamatex\\tamatex_state.db"

logging:
  level: "INFO"
  file: "./logs/tamatex.log"
  max_size_mb: 10
  backup_count: 5
```

#### 設定項目の説明

| 項目 | 説明 | 例 |
|------|------|-----|
| `nas.base_path` | NAS上のExcelフォルダのパス | `"Z:\\共有\\Excel"` |
| `nas.file_patterns` | 同期対象のファイルパターン | `["*.xlsx"]` |
| `nas.exclude_patterns` | 除外するファイルパターン | `["~$*", "*.tmp"]` |
| `google.credentials_path` | サービスアカウントJSONキーのパス | `"./config/service_account.json"` |
| `google.drive_folder_id` | 同期先のGoogle DriveフォルダID | `"1ABC..."` |
| `google.share_with` | 共有先メールアドレスのリスト | `["user@example.com"]` |
| `sync.interval_minutes` | 同期間隔（分） | `15` |
| `sync.state_db_path` | 状態DBの保存先（空なら自動） | `""` |
| `logging.level` | ログレベル（INFO / DEBUG / WARNING） | `"INFO"` |
| `logging.file` | ログファイルの出力先 | `"./logs/tamatex.log"` |

> **設定ミスの場合**: 起動時に明確なエラーメッセージが表示されます。
> - `nas` セクションが無い場合: `設定ファイルに 'nas' セクションがありません`
> - `google` セクションが無い場合: `設定ファイルに 'google' セクションがありません`
> - 必須項目（`base_path` 等）が無い場合: `設定ファイルのパラメータが不正です`

### 4-8. ログディレクトリの作成

```bat
mkdir C:\tamatex\logs
```

### 4-9. 初回セットアップの実行（任意）

NAS上のExcelファイルに対応するスプレッドシートを事前に一括作成する場合は、初回セットアップスクリプトを実行します。

```bat
cd C:\tamatex
.venv\Scripts\activate
python scripts\initial_setup.py
```

### 4-10. Windowsサービスの設定

手動インストールの場合、NSSM を使ってWindowsサービスとして登録します。

#### NSSM のインストール

1. https://nssm.cc/download にアクセス
2. 最新版をダウンロード（例: `nssm-2.24.zip`）
3. ZIPを展開し、`win64\nssm.exe` を `C:\tamatex\` にコピー

#### サービスの登録

コマンドプロンプトを **管理者として実行** し、以下を実行します。

```bat
cd C:\tamatex
nssm install tamatex
```

NSSM の設定画面が開きます。以下を入力してください。

**Application タブ:**

| 項目 | 値 |
|------|-----|
| Path | `C:\tamatex\.venv\Scripts\python.exe` |
| Startup directory | `C:\tamatex` |
| Arguments | `-m tamatex.main -c config\config.yaml` |

**Details タブ:**

| 項目 | 値 |
|------|-----|
| Display name | `tamatex - Excel Sync` |
| Description | `NAS Excel to Google Spreadsheet auto sync` |
| Startup type | `Automatic (Delayed Start)` |

> `Automatic (Delayed Start)` にすると、PC起動時にネットワーク接続が確立されてからサービスが開始されます。

**Exit actions タブ:**

| 項目 | 値 |
|------|-----|
| Restart action | `Restart application` |
| Restart delay | `60000` (60秒) |

**「Install service」** をクリック。

#### サービスの開始

```bat
nssm start tamatex
```

`SERVICE_RUNNING` と表示されればOKです。

**次のステップ**: [5. 動作確認](#5-動作確認) に進んでください。

```
2026-03-17 10:05:06 [INFO] tamatex - 新規同期完了: 在庫表 → xxxxxxxx
...
2026-03-17 10:08:00 [INFO] tamatex - --- 同期サイクル完了: スキャン=15, 同期=15, スキップ=0, エラー=0 ---
2026-03-17 10:08:00 [INFO] tamatex - 次回同期まで 15分 待機...
```

> **ログ項目の意味:**
> - **スキャン**: NAS上で検出されたExcelファイル数
> - **同期**: 今回同期（アップロード）されたファイル数
> - **スキップ**: 変更がなく同期不要だったファイル数
> - **エラー**: 同期に失敗したファイル数

### 5-3. スプレッドシートの確認

1. Google Drive の同期先フォルダ（例:「NAS同期データ」）を開く
2. `[同期] ファイル名` という名前のスプレッドシートを開く
3. NAS上のExcelと同じデータが表示されていればOK

> **結合セル対応**: Excelの結合セルは、結合範囲全体に左上セルの値が展開されてスプレッドシートに反映されます。

### 5-4. 更新の確認

1. NAS上のExcelファイルを開き、適当なセルを編集して保存
2. 15分後（次の同期サイクル）にスプレッドシートに反映されることを確認

### 5-5. 停止

**手動起動の場合:** コマンドプロンプトで `Ctrl + C` を押すと安全に停止します。

```
2026-03-17 10:20:00 [INFO] tamatex - シャットダウンシグナル受信 (signal=2)
2026-03-17 10:20:00 [INFO] tamatex - === tamatex 正常終了 ===
```

**サービスの場合:**

```bat
nssm stop tamatex
```

## 6. 営業・役員へのスプレッドシート共有

### 自動共有（推奨）

`config.yaml` の `share_with` にメールアドレスを記載すると、新しいスプレッドシート作成時に自動で共有されます。

```yaml
google:
  share_with:
    - "tanaka@example.com"
    - "suzuki@example.com"
    - "shacho@example.com"
```

> GUIインストーラーで導入した場合、ステップ5で入力したメールアドレスが自動設定されています。

### 手動共有

既に作成済みのスプレッドシートに後から共有を追加する場合は、Google Drive上で直接設定します。

1. Google Drive の同期先フォルダを開く
2. スプレッドシートを右クリック →「共有」
3. メールアドレスを入力 → 権限を「閲覧者」に設定 →「送信」

### 閲覧用URLの共有

営業・役員にはスプレッドシートのURLをブックマークしてもらいます。

1. スプレッドシートを開く
2. ブラウザのアドレスバーからURLをコピー
3. メールやチャットで共有

> **URLは固定です。** データが更新されても同じURLからアクセスできます。ブックマークしておけば、常に最新データを閲覧できます。

## 7. 運用・トラブルシューティング

### 7-1. ログの確認

ログファイルは以下に出力されます。

```
C:\tamatex\logs\tamatex.log
```

テキストエディタまたはコマンドプロンプトで確認できます。

```bat
type C:\tamatex\logs\tamatex.log
```

最新のログのみ確認する場合:
```bat
powershell Get-Content C:\tamatex\logs\tamatex.log -Tail 50
```

> ログは自動ローテーションされます（デフォルト: 10MB × 5世代）。古いログは `tamatex.log.1`〜`tamatex.log.5` に保存されます。

### 7-2. よくあるエラーと対処法

#### 「設定ファイルが見つかりません」

```
FileNotFoundError: 設定ファイルが見つかりません: ./config/config.yaml
```

**原因:** `config/config.yaml` が作成されていない
**対処:** GUIインストーラーで導入した場合は自動生成されます。手動の場合は `config.example.yaml` から `config.yaml` を作成してください。

#### 「設定ファイルに 'nas' セクションがありません」

```
ValueError: 設定ファイルに 'nas' セクションがありません: ./config/config.yaml
```

**原因:** `config.yaml` に `nas:` セクションが記載されていない
**対処:** 設定ファイルのフォーマットを確認してください。[4-7. 設定ファイルの作成](#4-7-設定ファイルの作成)の例を参考にしてください。

#### 「NASパスが見つかりません」

```
[ERROR] NASパスが見つかりません: Z:\共有\Excel
```

**原因:** NASのネットワークドライブが切断されている
**対処:**
1. エクスプローラーでNASにアクセスできるか確認
2. ネットワークドライブが切断されている場合は再接続
3. NAS本体の電源・ネットワーク接続を確認

> tamatexはNAS切断を自動検知し、接続が回復した次の同期サイクルで自動的に再開します。

#### 「認証ファイルが見つかりません」

```
FileNotFoundError: 認証ファイルが見つかりません: ./config/service_account.json
```

**原因:** サービスアカウントのJSONキーファイルが配置されていない
**対処:**
1. `config/service_account.json` が存在するか確認
2. GUIインストーラーで導入した場合はステップ3でJSONファイルを選択したか確認
3. 必要に応じてGoogle Cloud ConsoleからJSONキーを再ダウンロード

#### 「Google API認証エラー」

```
google.auth.exceptions.RefreshError: ...
```

**原因:** サービスアカウントのJSONキーファイルの内容が不正、またはAPIが無効
**対処:**
1. ファイルの内容が正しいJSONか確認
2. Google Cloud ConsoleでSheets API / Drive APIが有効になっているか確認
3. 必要に応じてGoogle Cloud ConsoleからJSONキーを再発行

#### 「Permission denied / 403 エラー」

```
gspread.exceptions.APIError: 403
```

**原因:** サービスアカウントにスプレッドシートまたはフォルダの権限がない
**対処:**
1. Google Drive上でフォルダがサービスアカウントと共有されているか確認
2. 共有権限が「編集者」になっているか確認
3. Google Sheets API / Drive API が有効になっているか確認

#### 「ファイル読み取りエラー（スキップ）」

```
[WARNING] ファイル読み取りエラー（スキップ）: Z:\...\file.xlsx - [Errno 13] Permission denied
```

**原因:** Excelファイルが別のユーザーによって開かれている、またはロック中
**対処:** 通常は一時的な問題です。次の同期サイクル（15分後）で自動的に再試行されます。

#### 「同期が15分以上経っても始まらない」

**対処:**
1. サービスが動作しているか確認: `nssm status tamatex`
2. ログファイルを確認して最後の出力を確認
3. サービスを再起動: `nssm restart tamatex`

### 7-3. よく使うサービス操作コマンド

| 操作 | コマンド |
|------|---------|
| 開始 | `nssm start tamatex` |
| 停止 | `nssm stop tamatex` |
| 再起動 | `nssm restart tamatex` |
| 状態確認 | `nssm status tamatex` |
| 設定変更 | `nssm edit tamatex` |
| サービス削除 | `nssm remove tamatex confirm` |

### 7-4. Excelファイルの追加・削除

| 操作 | tamatexの動作 |
|------|---------------|
| NASに新しいExcelファイルを追加 | 次の同期サイクルで自動検知し、スプレッドシートを自動作成 |
| NASからExcelファイルを削除 | スプレッドシートはそのまま残る（データ保全のため） |
| Excelファイル名を変更 | 旧ファイルは削除扱い、新ファイル名で新規スプレッドシートが作成される |
| Excelにシートを追加/削除 | 次の同期でスプレッドシートのシート構成も自動追従 |

### 7-5. NSSMのインストール

NSSMはPythonスクリプトをWindowsサービスとして登録するツールです。

1. https://nssm.cc/download にアクセス
2. 最新版をダウンロード（例: `nssm-2.24.zip`）
3. ZIPを展開し、`win64\nssm.exe` を `C:\tamatex\` にコピー
4. コマンドプロンプトで `C:\tamatex\nssm.exe --version` を実行して動作確認

### 7-6. 設定変更後の反映

`config.yaml` を変更した場合は、サービスの再起動が必要です。

```bat
nssm restart tamatex
```

### 7-7. 同期状態のリセット

全ファイルを強制的に再同期したい場合は、状態データベースを削除してサービスを再起動します。

```bat
nssm stop tamatex
del C:\tamatex\config\tamatex_state.db
nssm start tamatex
```

> **状態DBの場所**: デフォルトでは設定ファイル（`config\config.yaml`）と同じディレクトリ（`C:\tamatex\config\`）に `tamatex_state.db` が作成されます。`config.yaml` で `sync.state_db_path` を指定している場合は、そのパスのファイルを削除してください。

> **注意**: 状態データベースを削除すると、既存のスプレッドシートとのマッピング情報が失われます。新しいスプレッドシートが作成され、古いスプレッドシートは手動でGoogle Driveから削除する必要があります。スプレッドシートのデータ自体は削除されません。

## 付録

### 付録A: ネットワークドライブの割り当て

UNCパス（`\\192.168.x.x\...`）よりドライブレター（`Z:\...`）の方が安定するため、ネットワークドライブの割り当てを推奨します。

1. エクスプローラーで「PC」を右クリック →「ネットワークドライブの割り当て」
2. ドライブレター（例: `Z:`）を選択
3. フォルダに NASのパスを入力（例: `\\192.168.1.100\shared`）
4. **「サインイン時に再接続する」にチェック**
5. 「完了」をクリック

> **重要**: 「サインイン時に再接続する」にチェックを入れないと、PC再起動後にドライブが切断され、tamatexがNASにアクセスできなくなります。

### 付録B: exe版インストーラーのビルド

配布用にGUIインストーラーをexe化する場合は、開発環境で以下を実行します。

```bat
cd C:\tamatex\scripts
build_installer.bat
```

ビルドが成功すると `dist\Excel同期セットアップ.exe` が生成されます。

> ビルドにはPyInstallerが必要です（自動インストールされます）。
