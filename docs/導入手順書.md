## 1. 前提条件

以下が揃っていることを確認してください。

| 項目 | 要件 |
|------|------|
| 常時稼働PC | Windows 10/11、ネットワーク接続あり |
| Python | バージョン 3.11 以上 |
| QNAP NAS | PCからネットワークドライブまたはUNCパスでアクセス可能 |
| Google Workspace | 管理者権限を持つアカウント（共有ドライブ推奨） |
| インターネット | 常時接続（Google API通信用） |

### Python のインストール確認

コマンドプロンプトを開き、以下を実行します。

```
python --version
```

`Python 3.11.x` 以上が表示されればOKです。
表示されない場合は https://www.python.org/downloads/ からインストールしてください。

> **重要**: インストール時に **「Add Python to PATH」にチェック** を入れてください。

### NAS接続の確認

エクスプローラーでNASのExcelファイルが保存されているフォルダにアクセスできることを確認します。

**ドライブレターでアクセスしている場合（推奨）:**
```
Z:\共有\Excel
```

**UNCパスでアクセスしている場合:**
```
\\Tamatex-nas8tb\共有フォルダ\Excel
```

> **Windowsサービスで動かす場合の注意**:
> サービスは `LocalSystem` などの非対話アカウントで動作するため、ログインユーザーの認証情報を共有できません。NAS（SMB）の認証情報は `config.yaml` の `nas.auth` セクションに明示的に設定する必要があります（手順4-7参照）。

## 2. 導入方法の選択

tamatexには2つの導入方法があります。

| 方法 | 対象 | 所要時間 | 概要 |
|------|------|----------|------|
| **A. GUIインストーラー（推奨）** | 全ユーザー | 約10〜15分 | 7ステップのウィザードに従うだけで完了 |
| **B. 手動インストール** | IT技術者向け | 約20〜30分 | コマンドラインで各手順を実行 |

**はじめての方は方法Aを推奨します。** GUIインストーラーはGoogle Cloud設定からサービス起動まで、すべてのステップをガイド付きで案内します。

> ⚠️ **GUIインストーラーには既知の不具合があります**（Phase 2 修正対象）。問題が発生した場合は方法B（手動インストール）に切り替えてください。

---

## 3. 方法A: GUIインストーラーで導入（推奨）

GUIインストーラーウィザードが、以下のすべてを案内・自動実行します。

- Google Cloud プロジェクトとAPI設定のガイダンス
- サービスアカウント作成のガイダンスとキーファイル選択
- Google Driveフォルダ設定のガイダンス
- NASパス・同期間隔などの構成入力
- ファイルコピー、仮想環境作成、パッケージインストールの自動実行
- 設定ファイル（config.yaml）の自動生成
- Windowsサービスとしての登録・起動

### 3-1. インストーラーの起動

tamatexフォルダ内の `セットアップ.bat` をダブルクリックして起動します。

```
tamatex\セットアップ.bat
```

> **exe版がある場合**: `Excel同期セットアップ.exe` をダブルクリックしてください（Pythonのインストールが別途必要です）。

ウィザード画面が表示されます。左側のサイドバーに全7ステップが表示され、現在のステップがハイライトされます。

### 3-2. ステップ1: ようこそ

ウィザードの概要が表示されます。

- **所要時間**: 約10〜15分（Google Cloud設定を含む）
- **できること**: NAS上のExcelファイル自動検出、スプレッドシート / PDF への定期同期

内容を確認し、**「次へ」** をクリックします。

### 3-3. ステップ2: Google Cloud設定

Google Drive APIを利用するために、Google Cloudプロジェクトを準備します。ウィザードの指示に従って、ブラウザ上で以下を実施してください。

**操作手順:**

1. ウィザード画面の **「Google Cloud Consoleを開く」** ボタンをクリック
2. Google Workspaceの管理者アカウントでログイン
3. 画面上部のプロジェクト選択ドロップダウンから **「新しいプロジェクト」** をクリック
4. プロジェクト名に `tamatex` と入力して **「作成」** をクリック
5. 作成したプロジェクトが選択された状態で、左メニューの **「APIとサービス」→「ライブラリ」** を開く
6. **Google Drive API** を有効化:
   - 検索バーで `Google Drive API` を検索 → **「有効にする」** をクリック

> **Sheets API は不要です**。本システムは Drive API のみで Excel→Sheets変換 / PDFエクスポート の両方を実行します。

**完了後**: ウィザード画面の **「APIの有効化が完了しました」** にチェックを入れ、**「次へ」** をクリックします。

### 3-4. ステップ3: サービスアカウント

サービスアカウントは、tamatexがGoogle APIにアクセスするための専用アカウントです。

**操作手順:**

1. ウィザード画面の **「認証情報ページを開く」** ボタンをクリック
2. ブラウザで認証情報ページが開く（先ほど作成したプロジェクトが選択されていることを確認）
3. **「+ サービスアカウントを作成」** をクリック
4. 以下を入力:
   - サービスアカウント名: `tamatex-sync`（わかりやすい任意の名前）
   - サービスアカウントID: 自動入力される
   - 説明: `Excel to Spreadsheet sync`（任意）
5. **「作成して続行」** をクリック
6. ロールの選択は **スキップ**（「続行」→「完了」をクリック）
7. 作成されたサービスアカウント `tamatex-sync` をクリック
8. **「キー」タブ** を開く
9. **「鍵を追加」→「新しい鍵を作成」** をクリック
10. キーのタイプ: **JSON** を選択 →「作成」
11. JSONファイルが自動的にダウンロードされる

> **重要**: このJSONファイルは認証情報です。第三者に渡さないでください。

**ウィザードに戻って:**

12. **「参照...」** ボタンをクリックし、ダウンロードしたJSONファイルを選択
13. サービスアカウントのメールアドレスが表示されたことを確認（例: `tamatex-sync@tamatex-xxxxx.iam.gserviceaccount.com`）
14. **「次へ」** をクリック

### 3-5. ステップ4: Google Drive設定

同期先のスプレッドシート・PDFを格納するGoogle Driveフォルダを準備します。

> **共有ドライブを推奨**: サービスアカウントの個人ドライブには容量制限・所有権の問題があります。Google Workspace の **共有ドライブ** 上にフォルダを作成し、サービスアカウントを「コンテンツ管理者」として招待することを強く推奨します。

**操作手順（共有ドライブの場合）:**

1. ウィザード画面の **「Google Driveを開く」** ボタンをクリック
2. 「共有ドライブ」を開く（無い場合は新規作成）
3. 共有ドライブ内に同期先フォルダを作成（例: `NAS同期データ`）
4. 作成したフォルダを開き、ブラウザのアドレスバーからフォルダIDをコピー

```
https://drive.google.com/drive/folders/1ABCdefGHIjklMNOpqrSTUvwxYZ
                                        ^^^^^^^^^^^^^^^^^^^^^^^^^^^^
                                        この部分がフォルダID
```

5. 共有ドライブの「メンバーを管理」を開き、サービスアカウントのメールアドレスを **「コンテンツ管理者」** として追加

> **重要**: この共有設定を行わないと、サービスアカウントがフォルダにファイルを作成できません（403エラー）。

**ウィザードに戻って:**

6. **「フォルダID」** 欄にコピーしたIDを貼り付け
7. **「フォルダの共有設定が完了しました」** にチェックを入れる
8. **「次へ」** をクリック

> **同期先フォルダの構造**: 初回同期時に、指定フォルダ配下に自動的に `Sheets/` および `PDF/` のサブフォルダが作成されます。

### 3-6. ステップ5: 構成入力

同期設定とインストール先を入力します。

| 入力項目 | 説明 | 入力例 |
|----------|------|--------|
| **NASパス** | NAS上のExcelフォルダのパス | `Z:\共有\Excel` または UNCパス |
| **インストール先** | tamatexをインストールするフォルダ | `C:\nas-googlesync` |
| **同期間隔** | 同期の実行間隔（分） | `15`（デフォルト） |
| **共有メールアドレス** | スプレッドシートを共有する営業・役員のメールアドレス（1行1アドレス） | `eigyo@example.com` |

**NASパスの入力:**
- 「参照...」ボタンでフォルダ選択ダイアログから選択できます
- NASが接続されていない場合は、パスを手入力して「はい」で続行できます

**共有メールアドレス:**
- 1行に1つのメールアドレスを入力
- 空欄のままでもOK（後から `config.yaml` に追加可能）

入力が完了したら **「次へ」** をクリックします。

### 3-7. ステップ6: インストール

**「インストール開始」** ボタンをクリックすると、以下の処理が自動実行されます。

| ステップ | 処理内容 | 所要時間 |
|----------|----------|----------|
| [1/7] | インストールディレクトリ作成 | 数秒 |
| [2/7] | プロジェクトファイルのコピー | 数秒 |
| [3/7] | 設定ファイル（config.yaml）生成 | 数秒 |
| [4/7] | サービスアカウントキーのコピー | 数秒 |
| [5/7] | Python仮想環境の作成 | 約30秒 |
| [6/7] | tamatexパッケージのインストール | 約1〜3分 |
| [7/7] | ログディレクトリの確認 | 数秒 |

進捗バーとログが画面に表示されます。すべて緑色の「完了」表示になるまでお待ちください。

> **注意**: インストール中はウィザードを閉じないでください。

エラーが発生した場合は、ログ欄に赤色でエラー内容が表示されます。内容を確認し、問題を解消してから再試行してください。

### 3-8. ステップ7: 完了

インストールが成功すると、以下が表示されます。

- **インストール概要**: 設定内容のサマリー（インストール先、NASパス、同期間隔など）
- **同期サービスの開始ボタン**: クリックするとNSSMでWindowsサービスとして登録・起動します

**同期サービスの開始:**

1. **「同期サービスを開始」** ボタンをクリック
2. NSSMがインストールされている場合 → 自動的にサービス登録・起動
3. NSSMが見つからない場合 → 手動実行用のコマンドが表示されます

> NSSMのインストール方法は[7-5. NSSMのインストール](#7-5-nssmのインストール)を参照してください。

**「閉じる」** ボタンでウィザードを終了します。

**次のステップ**: [5. 動作確認](#5-動作確認) に進んでください。

## 4. 方法B: 手動で導入（上級者向け）

コマンドラインに慣れたIT技術者向けの手動導入手順です。

> **方法AのGUIインストーラーで導入済みの場合、この章はスキップしてください。**

### 4-1. Google Cloud プロジェクトの準備

1. https://console.cloud.google.com/ にアクセス
2. 新しいプロジェクトを作成（名前: `tamatex`）
3. **Google Drive API のみ** を有効化（Sheets API は不要）

### 4-2. サービスアカウントの作成

1. 「APIとサービス」→「認証情報」→「＋認証情報を作成」→「サービスアカウント」
2. 名前: `tamatex-sync` で作成
3. 「キー」タブ → JSON形式でキーをダウンロード
4. ダウンロードしたファイルの名前を `service_account.json` に変更

JSONファイルを開き、`client_email` の値を控えてください（次のステップで使用）。

```json
{
  "client_email": "tamatex-sync@tamatex-xxxxx.iam.gserviceaccount.com"
}
```

### 4-3. Google Drive フォルダの準備

1. https://drive.google.com/ で **共有ドライブ** に同期先フォルダを作成（例: `NAS同期データ`）
2. URL からフォルダIDを取得
3. 共有ドライブの「メンバーを管理」でサービスアカウントを **「コンテンツ管理者」** として追加

### 4-4. プロジェクトファイルの配置

tamatex フォルダを常時稼働PCの任意の場所にコピーします。

```
推奨配置例: C:\nas-googlesync\
```

### 4-5. サービスアカウントキーの配置

ダウンロードした `service_account.json` を以下に配置します。

```
C:\nas-googlesync\config\service_account.json
```

### 4-6. Python 仮想環境の作成とパッケージインストール

コマンドプロンプトを **管理者として実行** し、以下を順に実行します。

```bat
cd C:\nas-googlesync

python -m venv .venv

.venv\Scripts\activate

pip install .
```

> **重要**: `pip install -r requirements.txt` ではなく **`pip install .`** を使用してください。`pip install .` は依存パッケージに加えてtamatexパッケージ本体もインストールし、`tamatex` コマンドが使えるようになります。

以下のようなメッセージが表示されれば成功です。

```
Successfully installed openpyxl-3.x.x google-api-python-client-2.x.x google-auth-2.x.x pyyaml-6.x.x tamatex-0.1.0
```

### 4-7. 設定ファイルの作成

```bat
cd C:\nas-googlesync
copy config\config.example.yaml config\config.yaml
```

`config\config.yaml` をテキストエディタ（メモ帳等）で開き、以下を編集します。

```yaml
nas:
  # ★ NASのExcelフォルダパスを入力
  # Windowsパスの場合、バックスラッシュは二重にする
  base_path: "\\\\Tamatex-nas8tb\\共有フォルダ\\Excel"
  file_patterns:
    - "*.xlsx"
  exclude_patterns:
    - "~$*"
    - "*.tmp"
    - ".~lock*"
  # ★ Windowsサービスとして動かす場合は必須。
  # サービスは LocalSystem 等で動作し、対話ログインのcredentialキャッシュを共有しないため、
  # ここに NAS の SMB 認証情報を明示的に設定する。
  auth:
    server: "Tamatex-nas8tb"
    username: "admin"
    password: "your-nas-password"

google:
  # ★ サービスアカウントキーのパス（手順4-5で配置した場所）
  credentials_path: "./config/service_account.json"
  # ★ 手順4-3で控えたGoogle DriveフォルダID
  drive_folder_id: "1ABCdefGHIjklMNOpqrSTUvwxYZ"
  # ★ スプレッドシートを共有するメールアドレス
  share_with:
    - "eigyo@example.com"
    - "yakuin@example.com"

sync:
  # スケジュール方式（interval = N分間隔 / times = 指定時刻）
  mode: interval
  # interval モード時の同期間隔（分）
  interval_minutes: 15
  # times モード時の指定時刻リスト（HH:MM）
  # mode: times に切替時のみ使用。例:
  #   mode: times
  #   times:
  #     - "12:00"
  #     - "15:00"
  # 状態DBファイルのパス（空なら設定ファイルと同じディレクトリに自動作成）
  # state_db_path: "C:\\nas-googlesync\\config\\tamatex_state.db"

logging:
  level: "INFO"
  file: "./logs/tamatex.log"
  max_size_mb: 10
  backup_count: 5
```

#### 設定項目の説明

| 項目 | 説明 | 例 |
|------|------|-----|
| `nas.base_path` | NAS上のExcelフォルダのパス | `"\\\\server\\share\\folder"` |
| `nas.file_patterns` | 同期対象のファイルパターン | `["*.xlsx"]` |
| `nas.exclude_patterns` | 除外するファイルパターン | `["~$*", "*.tmp"]` |
| `nas.auth.server` | NAS の SMB サーバー名（UNCのバックスラッシュは不要） | `"Tamatex-nas8tb"` |
| `nas.auth.username` | NAS の SMB ユーザー名 | `"admin"` |
| `nas.auth.password` | NAS の SMB パスワード | `"xxxx"` |
| `google.credentials_path` | サービスアカウントJSONキーのパス | `"./config/service_account.json"` |
| `google.drive_folder_id` | 同期先のGoogle DriveフォルダID | `"1ABC..."` |
| `google.share_with` | 共有先メールアドレスのリスト | `["user@example.com"]` |
| `sync.mode` | スケジュール方式 (`interval` or `times`) | `"interval"` |
| `sync.interval_minutes` | (interval時) 同期間隔（分） | `15` |
| `sync.times` | (times時) 同期する時刻のリスト (`HH:MM`) | `["12:00", "15:00"]` |
| `sync.state_db_path` | 状態DBの保存先（空なら自動） | `""` |
| `logging.level` | ログレベル（INFO / DEBUG / WARNING） | `"INFO"` |
| `logging.file` | ログファイルの出力先 | `"./logs/tamatex.log"` |

> **設定ミスの場合**: 起動時に明確なエラーメッセージが表示されます。
> - `nas` セクションが無い場合: `設定ファイルに 'nas' セクションがありません`
> - `google` セクションが無い場合: `設定ファイルに 'google' セクションがありません`
> - 必須項目（`base_path` 等）が無い場合: `設定ファイルのパラメータが不正です`

> **セキュリティ**: `config.yaml` には NAS パスワードが平文で含まれます。Windows ACL で `Administrators` および `SYSTEM` のみアクセス可能に制限することを推奨します（`icacls` コマンド）。

### 4-8. ログディレクトリの作成

```bat
mkdir C:\nas-googlesync\logs
```

### 4-9. 初回同期の確認

`tamatex` を起動すると、初回サイクルで以下が自動実行されます。

- 同期先フォルダ配下に `Sheets/` および `PDF/` サブフォルダを自動作成
- NAS 上の各 Excel ファイルを Sheets 形式で `Sheets/` にアップロード
- 各 Sheets を PDF 化して `PDF/` に保存
- 状態DB（`tamatex_state.db`）を初期化

専用の初回セットアップスクリプトは不要です。サービス起動時に自動的に実行されます。

### 4-10. Windowsサービスの設定

NSSM を使ってWindowsサービスとして登録します。

#### NSSM のインストール

1. https://nssm.cc/download にアクセス
2. 最新版をダウンロード（例: `nssm-2.24.zip`）
3. ZIPを展開し、`win64\nssm.exe` を `C:\nas-googlesync\bin\` にコピー

#### サービスの登録

コマンドプロンプトを **管理者として実行** し、以下を実行します。

```bat
cd C:\nas-googlesync
bin\nssm.exe install tamatex
```

NSSM の設定画面が開きます。以下を入力してください。

**Application タブ:**

| 項目 | 値 |
|------|-----|
| Path | `C:\nas-googlesync\.venv\Scripts\python.exe` |
| Startup directory | `C:\nas-googlesync` |
| Arguments | `-m tamatex.main -c config\config.yaml` |

**Details タブ:**

| 項目 | 値 |
|------|-----|
| Display name | `tamatex - Excel Sync` |
| Description | `NAS Excel to Google Sheets/PDF auto sync` |
| Startup type | `Automatic` |

**Log on タブ:**

| 項目 | 値 |
|------|-----|
| Log on as | `Local System account` |

> NAS認証は `config.yaml` の `nas.auth` セクションでアプリ自身が `net use` を実行するため、`Local System account` で動作可能です。

**Exit actions タブ:**

| 項目 | 値 |
|------|-----|
| Restart action | `Restart application` |
| Restart delay | `30000` (30秒) |

**「Install service」** をクリック。

#### サービスの開始

```bat
bin\nssm.exe start tamatex
```

`SERVICE_RUNNING` と表示されればOKです。

**次のステップ**: [5. 動作確認](#5-動作確認) に進んでください。

## 5. 動作確認

### 5-1. サービスの状態確認

**NSSMでサービス登録した場合:**

```bat
bin\nssm.exe status tamatex
```

## 6. 営業・役員へのスプレッドシート共有

### 自動共有（推奨）

`config.yaml` の `share_with` にメールアドレスを記載すると、毎回の同期サイクルでスプレッドシート・PDFの両方に対し、閲覧者権限が **冪等に** 適用されます（既に共有済みなら何もしない）。

```yaml
google:
  share_with:
    - "tanaka@example.com"
    - "suzuki@example.com"
    - "shacho@example.com"
```

> GUIインストーラーで導入した場合、ステップ5で入力したメールアドレスが自動設定されています。
> 設定変更後は `bin\nssm.exe restart tamatex` でサービス再起動してください。

### 手動共有

既存ファイルに後から共有を追加する場合は、Google Drive上で直接設定します。

1. Google Drive の同期先フォルダを開く
2. `Sheets/` または `PDF/` 内のファイルを右クリック →「共有」
3. メールアドレスを入力 → 権限を「閲覧者」に設定 →「送信」

### 閲覧用URLの共有

営業・役員にはスプレッドシート・PDF のURLをブックマークしてもらいます。

1. Sheets または PDF を開く
2. ブラウザのアドレスバーからURLをコピー
3. メールやチャットで共有

> **URLは固定です。** データが更新されても同じURLからアクセスできます。

### 閲覧者がフィルタ・検索を使う

閲覧者権限のままで以下の操作が可能です。

- **検索**: Ctrl+F
- **フィルタビュー作成**: スプレッドシートで「データ」メニュー →「フィルタビューを作成」（自分専用、他の閲覧者には影響しません）

## 7. 運用・トラブルシューティング

### 7-1. ログの確認

ログファイルは以下に出力されます。

```
C:\nas-googlesync\logs\tamatex.log
```

最新のログのみ確認する場合:
```bat
powershell Get-Content C:\nas-googlesync\logs\tamatex.log -Tail 50 -Encoding UTF8
```

> ログは自動ローテーションされます（デフォルト: 10MB × 5世代）。古いログは `tamatex.log.1`〜`tamatex.log.5` に保存されます。

### 7-2. よくあるエラーと対処法

#### 「設定ファイルが見つかりません」

```
FileNotFoundError: 設定ファイルが見つかりません: ./config/config.yaml
```

**原因:** `config/config.yaml` が作成されていない、またはサービスのワーキングディレクトリが正しくない
**対処:**
- 手動の場合は `config.example.yaml` から `config.yaml` を作成
- サービス実行時は NSSM の `Startup directory` が `C:\nas-googlesync` になっているか確認

#### 「NAS認証失敗」

```
RuntimeError: NAS認証失敗: server=xxx user=xxx (net use returncode=2)
```

**原因:** `nas.auth` セクションのユーザー名・パスワードが誤っている、またはNASが応答しない
**対処:**
1. ユーザー名・パスワードを再確認（大文字小文字も含めて）
2. NASのアカウントロックアウトに注意（試行回数制限あり）
3. エクスプローラーから同じ認証情報でアクセスできるか手動確認

#### 「NASパスが見つかりません」

```
[ERROR] NAS接続エラー（次回サイクルで再試行）: NASパスが見つかりません: ...
```

**原因:** NASの電源・ネットワーク・認証のいずれかが切れている
**対処:**
1. NAS本体の電源・ネットワーク接続を確認
2. `nas.auth` の認証情報が正しいか確認
3. tamatexは自動再試行しますので、復旧後は次の同期サイクルで自動再開

#### 「認証ファイルが見つかりません」

```
FileNotFoundError: 認証ファイルが見つかりません: ./config/service_account.json
```

**原因:** サービスアカウントのJSONキーファイルが配置されていない
**対処:**
1. `config/service_account.json` が存在するか確認
2. 必要に応じてGoogle Cloud ConsoleからJSONキーを再ダウンロード

#### 「Google API認証エラー」

```
google.auth.exceptions.RefreshError: ...
```

**原因:** サービスアカウントのJSONキーファイルの内容が不正、またはAPIが無効
**対処:**
1. ファイルの内容が正しいJSONか確認
2. Google Cloud Console で **Drive API** が有効になっているか確認
3. 必要に応じてJSONキーを再発行

#### 「Permission denied / 403 エラー」

```
googleapiclient.errors.HttpError: 403 ... Forbidden
```

**原因:** サービスアカウントが Drive フォルダにアクセスできない
**対処:**
1. 共有ドライブのメンバー設定でサービスアカウントが「コンテンツ管理者」として登録されているか確認
2. 通常の Drive フォルダに置いている場合は、フォルダの共有設定でサービスアカウントを編集者として追加
3. Drive APIが有効になっているか確認

#### 「Drive storage quota has been exceeded」

```
googleapiclient.errors.HttpError: 403 ... Drive storage quota has been exceeded
```

**原因:** サービスアカウントが個人ドライブにファイルを作成しようとしている（個人ドライブの容量がない）
**対処:** 共有ドライブにフォルダを作成し、サービスアカウントを「コンテンツ管理者」として招待してください。

#### 「ファイル読み取りエラー（スキップ）」

```
[WARNING] 同期失敗（スキップ）: xxx.xlsx - [Errno 13] Permission denied
```

**原因:** Excelファイルが別のユーザーによって開かれている、またはロック中
**対処:** 通常は一時的な問題です。次の同期サイクル（15分後）で自動的に再試行されます。

#### 「同期が15分以上経っても始まらない」

**対処:**
1. サービスが動作しているか確認: `bin\nssm.exe status tamatex`
2. ログファイルを確認して最後の出力を確認
3. サービスを再起動: `bin\nssm.exe restart tamatex`

### 7-3. よく使うサービス操作コマンド

| 操作 | コマンド |
|------|---------|
| 開始 | `bin\nssm.exe start tamatex` |
| 停止 | `bin\nssm.exe stop tamatex` |
| 再起動 | `bin\nssm.exe restart tamatex` |
| 状態確認 | `bin\nssm.exe status tamatex` |
| 設定変更 | `bin\nssm.exe edit tamatex` |
| サービス削除 | `bin\nssm.exe remove tamatex confirm` |

### 7-4. Excelファイルの追加・削除

| 操作 | tamatexの動作 |
|------|---------------|
| NASに新しいExcelファイルを追加 | 次の同期サイクルで自動検知し、Sheets と PDF を自動作成 |
| NASからExcelファイルを削除 | Sheets / PDF はそのまま残る（データ保全のため） |
| Excelファイル名を変更 | 旧ファイルは削除扱い、新ファイル名で新規 Sheets / PDF が作成される |
| Excelにシートを追加/削除 | 次の同期で Sheets 側も完全一致するよう更新される |

### 7-5. NSSMのインストール

NSSMはPythonスクリプトをWindowsサービスとして登録するツールです。

1. https://nssm.cc/download にアクセス
2. 最新版をダウンロード（例: `nssm-2.24.zip`）
3. ZIPを展開し、`win64\nssm.exe` を `C:\nas-googlesync\bin\` にコピー
4. コマンドプロンプトで `C:\nas-googlesync\bin\nssm.exe --version` を実行して動作確認

### 7-6. 設定変更後の反映

`config.yaml` を変更した場合は、サービスの再起動が必要です。

```bat
bin\nssm.exe restart tamatex
```

### 7-7. 同期状態のリセット

全ファイルを強制的に再同期したい場合は、状態データベースを削除してサービスを再起動します。

```bat
bin\nssm.exe stop tamatex
del C:\nas-googlesync\config\tamatex_state.db
bin\nssm.exe start tamatex
```

> **状態DBの場所**: デフォルトでは設定ファイル（`config\config.yaml`）と同じディレクトリ（`C:\nas-googlesync\config\`）に `tamatex_state.db` が作成されます。`config.yaml` で `sync.state_db_path` を指定している場合は、そのパスのファイルを削除してください。

> **挙動**: 状態DBを削除すると、tamatex は全ファイルを「未同期」と判定して再アップロードします。
> 既存の Drive 上のファイル名と一致するスプレッドシート・PDFは **fileId が新しいものに置き換わる** 場合がある（古いファイルが残ったままになる）ため、必要に応じて Drive 上の古いファイルを手動削除してください。

## 付録

### 付録A: ネットワークドライブの割り当て

サービス実行時はドライブレターよりUNCパスが安定します（サービスはユーザーセッションのドライブマップを引き継がないため）。本システムでは `nas.auth` セクションで認証情報を渡すことで、UNCパス（`\\server\share\...`）を直接利用することを推奨します。

ただし、対話ログイン時の動作確認のためにドライブレター割り当てを残す場合は以下のように設定できます。

1. エクスプローラーで「PC」を右クリック →「ネットワークドライブの割り当て」
2. ドライブレター（例: `Z:`）を選択
3. フォルダに NASのパスを入力（例: `\\Tamatex-nas8tb\共有フォルダ`）
4. **「サインイン時に再接続する」にチェック**
5. 「完了」をクリック

### 付録B: exe版インストーラーのビルド

配布用にGUIインストーラーをexe化する場合は、開発環境で以下を実行します。

```bat
cd C:\nas-googlesync\scripts
build_installer.bat
```

ビルドが成功すると `dist\Excel同期セットアップ.exe` が生成されます。

> ビルドにはPyInstallerが必要です（自動インストールされます）。
