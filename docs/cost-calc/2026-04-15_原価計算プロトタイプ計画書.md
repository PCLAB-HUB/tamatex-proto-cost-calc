# 原価計算プロトタイプ実装計画書

**作成日**: 2026-04-15
**プロジェクト**: tamatex — 輸入原価計算ウェブアプリ プロトタイプ
**目的**: 顧客への実現可能性デモ（モック品番による実動作）

---

## 1. 背景と目的

### 1.1 背景

2026-04-14の会議において、以下が確認された：

- 現行のExcel原価計算シートは98列にわたる複雑な構成で、新入社員が入力位置で迷い、計算式を破壊するリスクがある
- 各営業担当が個別のファイルで管理しており、計算方法もバラバラ
- 品番マスター不在により、在庫追跡・クレーム対応・ロット特定が困難
- 原価計算→見積書作成→受注確定の一貫したワークフローが存在しない

顧客は「本当にこれがウェブアプリで実現できるのか」を確認したい段階にある。

### 1.2 プロトタイプの目的

1. **Excelと同一の計算結果**が出ることを実証する（数値の一致を顧客が検証可能）
2. **ウェブUIで操作可能**であることを体感してもらう
3. **品番マスタとの連携イメージ**をモックで見せる
4. 本番開発の判断材料とする

### 1.3 プロトタイプの範囲

| 含む | 含まない |
|------|---------|
| 輸入原価計算の全計算ロジック | 品番マスタの正式運用 |
| モック品番6品目 + ギフト12パターン | 見積書PDF出力 |
| 20FT/40FTコンテナ条件切替 | ユーザー認証 |
| リアルタイム計算結果表示 | データ永続化（DB保存） |
| Excelとの計算結果比較機能 | メール送信連携 |
| マスタパラメータ編集UI | 仮/確定ステータス管理 |

## 2. Excel解析結果

### 2.1 計算の全体構造

```
┌──────────────────────────────────────────────────────────────┐
│ 【パラメータ層】                                              │
│  社内為替(150円), 現行為替(150円), ロス率(15%),               │
│  刺繍単価(4種), 保険率(0.18%), 関税率(7.4%),                 │
│  輸入経費(11項目), 物流経費(5項目)                            │
├──────────────────────────────────────────────────────────────┤
│ 【単品層】最大10品目                                          │
│  品名, アイテム種別, サイズ, 目方, FOB(USD), 加工費           │
│  → 製造原価(円) = FOB×為替 + 加工費 + 資材代 + 償却費        │
├──────────────────────────────────────────────────────────────┤
│ 【ギフトセット層】最大20パターン                               │
│  単品の組合せ(枚数指定) → タオル代                            │
│  + 資材代(化粧箱等7項目 × ロス率) + 償却費                   │
│  + 加工代(箱入等)                                            │
│  = FOB合計                                                   │
│  → C&F → CIF → 関税 → 輸入経費 → 物流                      │
│  = 製造原価                                                  │
│  → ×(1+マージン) = 見積単価                                  │
│  → 粗利 = 販売単価 - 製造原価                                │
├──────────────────────────────────────────────────────────────┤
│ 【条件切替】                                                  │
│  20FTコンテナ/大阪揚げ/今治倉庫 ←→ 40FTコンテナ/東京揚げ    │
│  （計算ロジック同一、パラメータのみ異なる）                     │
└──────────────────────────────────────────────────────────────┘
```

### 2.2 計算式仕様（全量）

#### 2.2.1 単品製造原価（上段・行7-16）

```
入力項目:
  C  = 品名
  E  = アイテム種別 (BT/FT/BM/MT)
  F  = サイズ (例: 60*120)
  G  = 目方(g)
  H  = 織区分 (JQ等)
  L  = ロット数
  P  = FOB単価(USD)
  R  = 刺繍針数(千針)
  S  = 銀糸刺繍(USD)
  T  = ケット刺繍(USD) ※型代をロットで割る: T/L
  U  = ブランドロゴ(USD)
  V  = 検品代(円)
  W  = その他(円)
  X  = ネーム1(円)
  Y  = ネーム2(円)
  Z  = シール(円)
  AA = その他2(円)
  AQ = トレス単価(円)
  AR = 型数
  AT〜BA = 償却費項目（図案、JQカード、刺繍カード、見本代、検査費用、資材版×3）

計算式:
  AB  = (X + Y + Z + AA) × (1 + ロス率)               ... 日本支給資材
  AS  = AQ × AR                                        ... トレス小計
  BC  = AT + AU + AV + AS + AW + AX + AY + AZ + BA     ... 償却費合計
  BB  = BC / L                                          ... 償却費/枚
  BE  = (P + (R/1000 × R4) + S + T/L + U) × 社内為替 + V + W + AB  ... FOB(円)
  BZ  = 3,200,000 / G                                  ... 積載個数（20FT基準容積÷目方）
  BY  = BZ / BX                                         ... 積載ケース数
  BG  = BE + (BF / BZ) × 現行為替                       ... C&F
  BI  = BG × (1 + 保険率)                               ... CIF
  BK  = ((P+(R/1000)×R4+S+T/L+U+BF/BZ)×現行為替+V+W+AB) × 関税率  ... 関税
  CA  = BO×現行為替 + BM+BN+BP+BQ+BR+BS+BT+BU+BV+BW   ... 輸入経費合計(コンテナ)
  CB  = CA / BZ                                         ... 輸入経費単価
  CP  = (CI + CJ + CK + CL + CM×CN) / CO               ... 物流経費単価
  M   = BE + V + W + AB + BB                            ... 製造原価(円)
  N   = BI + BK + CB + CP + BB                          ... 円建原価(別計算)
```

#### 2.2.2 ギフトセット原価（下段・行23-42）

```
入力項目:
  C   = セット名
  D   = 上代
  E   = 販売単価
  I   = 販売予定数量
  O   = 歩引（カタログ掲載料率）
  R-AA = 単品①〜⑩の使用枚数
  AO  = 化粧箱(円)
  AP  = ブランドラベル(円)
  AQ  = 台紙(円)
  AR  = OP(円)
  AS  = 保護紙(円)
  AT  = JAN/シール(円)
  AU  = その他資材(円)
  AV  = デザイン費(円) ← 償却
  AW  = 木型(円) ← 償却
  AX  = 版代(円) ← 償却
  AY  = その他償却(円)
  BX  = ケース入数(個)
  BY  = 積載ケース数
  CD  = 箱入代(円)
  CE  = その他加工1(円)
  CF  = その他加工2(円)
  CI  = 運賃(ケース)(円)
  CJ  = 梱包(円)
  CK  = その他(円)

共有パラメータ（下段全体で共通）:
  N20 = マージン(%) → N22 = N20/100
  BA20 = 資材ロット
  BB20 = 資材ロス率(%) → BB22 = BB20/100
  R22〜AA22 = 単品①〜⑩の製造原価(円) ← 上段M7〜M16を参照
  BF22 = 海外運賃(USD)
  BH22 = 保険率
  BJ22 = 関税率
  BM22〜BW22 = 輸入経費11項目

計算式:
  AB  = Σ(単品i製造原価 × 使用枚数i), i=①〜⑩         ... タオル代
  ADi = 数量(I) × 使用枚数(Ri)                         ... 分解数（単品ごと）
  AZ  = (AV + AW + AX + AY) / BA                       ... 償却単価
  BC  = (AO+AP+AQ+AR+AS+AT+AU) × (1+資材ロス率) + AZ   ... 資材代計
  CG  = CD + CE + CF                                    ... 加工代計
  BE  = AB + BC + CG                                    ... FOB合計
  BZ  = BX × BY                                         ... 積載個数
  BG  = BE + (BF / BZ) × 現行為替                       ... C&F
  BI  = BG × (1 + 保険率)                               ... CIF
  BK  = BI × 関税率                                     ... 関税
        ※注: 単品層とギフト層で関税計算式が異なる
  CA  = BO×社内為替 + BM+BN+BP+BQ+BR+BS+BT+BU+BV+BW   ... 輸入経費合計
        ※注: 単品層はBO×現行為替、ギフト層はBO×社内為替
  CB  = CA / BZ                                         ... 輸入経費単価
  CO  = BX                                              ... 物流の入数
  CP  = (CI + CJ + CK + CL + CM×CN) / CO               ... 物流経費単価
  P   = BI + BK + CB + CP + E×O                         ... 製造原価（+歩引）
  M   = ROUNDUP(P × (1 + マージン率), 0)                ... 見積単価
  F   = E / D                                           ... 上代掛率
  G   = E - P                                           ... 粗利単価
  H   = (E - P) / E                                     ... 粗利率
  J   = D × I                                           ... 上代金額
  K   = E × I                                           ... 売上金額
  L   = K × H                                           ... 粗利金額

集計行（行22）:
  I22 = SUM(I23:I42)                                    ... 総数量
  K22 = SUM(K23:K42)                                    ... 総売上
  L22 = SUM(L23:L42)                                    ... 総粗利
  H22 = L22 / K22                                       ... 平均粗利率
```

#### 2.2.3 20FTと40FTの差異

```
                    20FT大阪/今治        40FT東京
マージン(%)         40                    15
海外運賃(USD)       160                   240
THC                 ¥36,000               ¥55,000
CIC                 ¥180                  ¥0
EMC                 ¥3,000                ¥46,000
CIC2                ¥20,000               ¥4,000
D/O                 ¥5,000                ¥3,000
ドレー料            ¥51,000               ¥48,000
デバン料            ¥0                    ¥0
入出庫              ¥70                   ¥140
保管料              ¥120                  ¥200
積載ケース(BY)      150                   可変(BY列で個別設定)

40FT固有: ギフトの構成・資材・加工代は20FTから参照（=C23等）
40FT固有: 歩引の計算で CP→CI に変わっている（P49式）
40FT固有: AZ49に#REF!エラーあり（$BA$20 → #REF! 壊れた参照）
```

### 2.3 発見した問題点

| # | 問題 | 影響 | 対応方針 |
|---|------|------|---------|
| 1 | 40FT行のAZ49に#REF!エラー | 償却費がゼロになる | BA20(=3000)を参照するように修正 |
| 2 | 単品行8,11,12のN列(円建原価)がゼロ | 目方(G列)が未入力→BZ=0→除算エラー | 目方未入力時はスキップ |
| 3 | 40FTのP49式でCB→CIに変わっている | 20FTとロジックが異なる可能性 | 両方実装し、20FT結果で検証 |
| 4 | ギフトBE49式で`AB49`→`AA49`に変わっている | タオル代ではなく最後の単品枚数を参照 | 20FT式(AB+BC+CG)を正とする |
| 5 | ロス率の二重定義（Z4=15%, AB4=0.15） | AB4はZ4/100の計算式 | 内部的にはAB4(0.15)を使用 |

## 4. 技術設計

### 4.1 技術スタック

```
Streamlit 1.x          — UI フレームワーク
Python 3.11+            — 計算エンジン
dataclasses             — データモデル
pytest                  — Excel値との照合テスト
```

**Streamlit選定理由**:
- Python単体で完結（追加のフロントエンド開発不要）
- フォーム入力 → リアルタイム計算表示が得意
- 顧客デモとして十分な見栄え
- 計算エンジン（純Python関数）は本番アプリにそのまま移植可能

## 6. 重要な設計判断

### 6.1 計算精度

- Pythonのfloatで計算（Excelと同じIEEE 754倍精度）
- ROUNDUP関数は `math.ceil` で再現
- 検証許容誤差: ±0.01円（浮動小数点の丸め差異）

### 6.2 単品行の目方なしケース

Excel上、目方(G列)が空の単品（②⑤⑥）はBZ列（積載個数）がゼロになり、C&F以降の計算がゼロになる。これはExcelの`IFERROR`で0に丸められている。

**プロトタイプでの対応**: 目方なし=C&F以降ゼロとして再現（Excelと同じ挙動）

### 6.3 関税計算式の単品/ギフトの差異

- **単品**: `BK = ((FOB_USD + 加工 + 海外運賃/積載数) × 現行為替 + 国内加工) × 関税率`
  - 関税の課税対象にFOBだけでなく海外運賃按分と国内加工費を含む
- **ギフト**: `BK = CIF × 関税率`
  - CIF全額に関税率を乗じる（よりシンプル）

両方とも忠実に実装する。

### 6.4 40FTの#REF!エラー

40FT行のAZ49に`#REF!`がある。プロトタイプでは20FTと同じ`BA20`（=3000）を使用する。この判断は計画書に記録し、顧客に確認事項として提示する。

### 6.5 プロトタイプと本番の境界

| 項目 | プロトタイプ | 本番 |
|------|------------|------|
| 品番 | モック6品目ハードコード | 品番マスタDB |
| ギフト構成 | モック12パターン | 動的に構成 |
| 輸入条件 | 2条件（UI切替） | 条件マスタDB |
| 計算エンジン | **そのまま移植** | **そのまま移植** |
| データ永続化 | なし（セッション内） | PostgreSQL |
| 認証 | なし | Google OAuth等 |
| 見積書 | 表示のみ | PDF出力+メール |

## 7. リスクと対策

| # | リスク | 影響 | 対策 |
|---|--------|------|------|
| 1 | Excelの計算値と一致しない | デモの信頼性が下がる | Phase 1で全量テストケースを先に作り、差異を0にする |
| 2 | Excelの不明点（4箇所）が計算に影響 | 一部の値が不正確 | 現行Excelの値をそのまま使い、不明点リストを添付 |
| 3 | 40FTの壊れた式 | 40FT条件の結果が不正確 | 20FTの検証を優先し、40FTは「参考値」として提示 |

| 5 | 単品の追加・削除がUI上でできない | 柔軟性が低い | Phase 2で追加機能の余地を残す |
