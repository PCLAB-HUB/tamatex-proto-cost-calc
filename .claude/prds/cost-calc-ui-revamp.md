# PRD: cost-calc-ui-revamp

## Executive Summary

原価計算プロトタイプ（Streamlit）のUIを、**営業・役員・社長が意思決定に使えるダッシュボード型**に再構成する。現状は計算ロジック検証のための「Excel代替画面」であり、数字が並ぶだけで「高いのか安いのか」「為替が振れた時の影響」が直感的に分からない。本改修で、起動直後のトップビューにKPIカードと主要チャートを集約し、シナリオ（20FT vs 40FT、USD=150 vs 160 等）を名前付きで永続保存・横並び比較できるようにする。

技術スタックは Streamlit を継続し、`streamlit-aggrid` / `plotly` / `streamlit-extras` 等で視覚的完成度を引き上げる。既存の計算エンジン（`proto/engine/`、131テスト）には一切手を入れず、UI レイヤーのみを改修する。

## Problem Statement

### 現状
- アプリを開くと「基本情報」タブが表示されるが、**主要KPIが数字の羅列**で一目で把握できない
- サイドバー入力項目が多く（輸入条件11項目×2、為替・マージン・資材・物流）、**画面情報量が過剰**で目線が迷う
- **シナリオ保存機能がない** — 「先週見た試算結果」「20FT案 vs 40FT案」を後から呼び出せない
- **横並び比較ができない** — 条件を変えて再計算するたびに前の数字が消える
- 営業が商談中にタブレット/PCで「この品番の原価？」を確認するユースケースに対して、**5タブ・全サイドバー編集を経由**する必要がある

### 改修の動機
- 現場導入後、営業・役員が「結局Excelの方が早い」と離脱するリスク
- 顧客プレゼンで見せる際、**見た目のプロ感が足りない**
- シナリオ比較ができないと、経営判断の意思決定ツールとして機能しない

## User Stories

### US-1: 営業が商談中に即座に原価を確認
**As a** 営業担当（PC/タブレット使用）
**I want to** アプリを開いて3秒で「今の条件での平均原価・上代・粗利率」を把握したい
**So that** 商談中の質問に即答できる

**受入基準:**
- アプリ起動直後のトップ画面（ダッシュボード）に平均原価・平均上代・平均粗利率が大きく表示される
- PC・タブレット（ランドスケープ）で主要KPIが一画面に収まる
- 条件変更はサイドバーの最低限の項目（コンテナ・為替）だけで完結

### US-2: 役員が為替感度を把握
**As a** 役員
**I want to** 為替レートが±10円振れた時の原価への影響を視覚的に見たい
**So that** 為替ヘッジの意思決定ができる

**受入基準:**
- ダッシュボードに「USD 140〜160」の範囲で平均原価が変動するチャートが常時表示される
- チャート上に現在値（例: USD=150）のマーカーがある
- 品目別に最も為替影響が大きいものを特定できる（横棒または感度ランキング）

### US-3: 社長が複数シナリオを横並び比較
**As a** 社長
**I want to** 「標準シナリオ」と「新商品シナリオ」を2カラムで横並び比較したい
**So that** どちらが利益率高いか一目で判断できる

**受入基準:**
- シナリオ一覧からチェックボックスで2件選択 → 「比較」ボタンで比較ビューに遷移
- 比較ビューで、主要KPI（平均原価・上代・粗利率）と品目別原価が2カラム並びで表示される
- 差分（金額差・%差）がハイライトされる

### US-4: 営業がシナリオを名前付きで保存
**As a** 営業担当
**I want to** 「USD150_20FT_標準」などの名前を付けて現在の条件を保存したい
**So that** 次回同じ条件を再入力せずに済む

**受入基準:**
- サイドバー下部に「現在の条件をシナリオとして保存」ボタン
- 名前入力ダイアログ → 保存成功のトースト通知
- シナリオ一覧から選択すると、サイドバーの全入力値が復元される
- ブラウザを閉じても消えない（SQLite永続化）
- シナリオは複製・リネーム・削除可能

## Functional Requirements

### FR-1. ダッシュボードタブ（新規、起動時デフォルト）

**構成要素**:
1. **ヘッダー帯**: 現在の条件サマリ（コンテナ名 / USD / マージン目標）
2. **KPIカード 3枚（横並び、モバイル時は縦積み）**:
   - 平均原価（円）— 単品6品目の平均、サブテキストで品目数
   - 平均上代（円）— 粗利率50%想定の上代、サブテキストで計算根拠
   - 平均粗利率（%）— 現在の上代設定時の粗利率、サブテキストで目標値との差
3. **為替感度チャート**: 横軸 USD(140〜160)、縦軸 平均原価、現在値マーカー付き
4. **品目別原価ランキング**: 横棒グラフ、6品目を原価昇順、差額ラベル表示
5. **ギフトセット サマリカード**: 最大粗利セット / 最小粗利セットを2カードで表示

### FR-2. シナリオ管理機能（新規）

**ストレージ**:
- `proto/data/scenarios.db`（SQLite）
- テーブル `scenarios(id, name, created_at, updated_at, condition_json)`
- `condition_json` は `ImportCondition` dataclass を JSON シリアライズ

**UI**:
- **サイドバー下部**: 「💾 現在の条件を保存」ボタン + 名前入力
- **新規タブ「シナリオ」**:
  - シナリオ一覧テーブル（aggrid）: 名前・更新日時・チェックボックス
  - 「📂 読み込み」ボタン: 1件選択時に条件復元
  - 「📋 複製」ボタン: 1件選択時に複製
  - 「✏️ リネーム」ボタン
  - 「🗑️ 削除」ボタン（確認ダイアログ）
  - 「📊 比較」ボタン: 2件選択時にモーダル or 新ビューで横並び比較
- **比較ビュー**:
  - 左右2カラムで KPI カード・品目別原価・ギフトセット結果を並置
  - 差分を `+¥123` / `-¥456` で色分け表示（緑=改善、赤=悪化）

**永続化**:
- アプリ起動時に `scenarios.db` を読み込み、存在しなければ自動生成
- シナリオ操作は即座に DB に反映（楽観的UI）

### FR-3. 既存タブの UI 刷新

- **タブ順序変更**: ダッシュボード → 単品 → ギフト → 比較一覧 → シナリオ → Excel検証（検証は最後尾）
- **テーブル刷新**: `st.dataframe` → `streamlit-aggrid` 採用
  - 通貨列は `¥1,234` 形式
  - 粗利率列はバーチャート付きセル
  - 列幅自動調整・ソート・フィルタ有効
- **サイドバー整理**:
  - 輸入経費11項目 × 2 は「上級者設定」としてデフォルト折りたたみ
  - 最頻使用項目（コンテナ・為替・マージン）を最上段に固定

### FR-4. サイドバー UX 改善

- アコーディオン（`st.expander`）でセクション折りたたみ
- 「シナリオから復元」時に折りたたみ状態も保持
- 入力値変更時に即計算（既存挙動維持）

## Non-Functional Requirements

### パフォーマンス
- シナリオ一覧読み込み: 100件までで 500ms 以内
- ダッシュボード初回描画: 2秒以内
- シナリオ保存: 200ms以内

### 互換性
- 既存の `proto/engine/` モジュールは**変更しない**
- 既存131テスト（計算ロジック）は全て PASS 継続
- Streamlit Cloud デプロイ環境で動作

### 保守性
- UI ロジックは `proto/ui/` 配下、新規: `proto/ui/dashboard.py`, `proto/ui/scenarios.py`
- シナリオ永続化は `proto/storage/scenario_repo.py` に分離（将来 Sheets 移行時の差し替えを容易に）
- dataclass ↔ JSON 変換ユーティリティは単一箇所に集約

### セキュリティ
- シナリオDBはローカルファイルのみ（認証不要）
- SQLインジェクション対策: パラメータバインド徹底
- `scenarios.db` は `.gitignore` に追加

## Constraints & Assumptions

### 制約
- **Streamlit 継続前提**: Next.js 等への刷新はスコープ外
- 認証・権限管理は**なし**（誰でも編集可能な前提）
- Streamlit Cloud の無料枠制限内で動作（メモリ・ビルド時間）
- 日本語フォント・通貨表示のブラウザ差異は考慮するが完全統一はしない

### 前提
- 顧客（社長）はPCまたはタブレットで閲覧が多いが、営業はスマホ利用もあり
- 既存 `proto/engine/` は仕様確定済み、UI層だけ変更すれば計算は保証される
- `streamlit-aggrid`, `plotly`, `streamlit-extras` の追加依存は許容
- `PCLAB-HUB/tamatex-proto-cost-calc` リポジトリを Streamlit Cloud デプロイ用に継続利用

## Out of Scope

- **モバイル対応（スマホ幅 〜600px）— 将来フェーズで再検討**
- 認証・ロール管理（誰が編集/閲覧できるか）
- Google Sheets 連携でのシナリオ共有（Lv.4、将来フェーズ）
- 品番マスタ連携
- 帳票生成（見積書・注文書等）— 別エピック予定
- ネイティブモバイルアプリ化
- リアルタイム協調編集
- 計算エンジンの追加変更（40FT 新規検証・新しい輸入経費パラメータ等）
- 多言語対応
