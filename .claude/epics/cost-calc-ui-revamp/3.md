# Task: シナリオ永続化層の実装

## Description

シナリオ（条件セット）を SQLite に永続化する `proto/storage/` パッケージを新規実装する。`ImportCondition`（ネストされた `ImportExpenses` を含む dataclass）を JSON シリアライズして保存し、CRUD 操作を提供する。

シナリオ管理機能（Lv.3）のバックエンドとなるため、**このタスクの品質がエピック全体の信頼性を決める**。ユニットテスト必須。

## Technical Details

### `proto/storage/serializer.py` 設計方針

```python
from dataclasses import asdict
import json
from proto.engine.models import ImportCondition, ImportExpenses


def condition_to_json(cond: ImportCondition) -> str:
    return json.dumps(asdict(cond), ensure_ascii=False)


def condition_from_json(s: str) -> ImportCondition:
    d = json.loads(s)
    # ネストされた dataclass を手動で復元
    d["import_expenses_single"] = ImportExpenses(**d["import_expenses_single"])
    d["import_expenses_gift"] = ImportExpenses(**d["import_expenses_gift"])
    return ImportCondition(**d)
```

**注意点**:
- `ImportCondition` の全フィールドを把握して、他にネストされた dataclass があれば追加対応
- 数値精度の劣化がないこと（`float` → JSON → `float` のラウンドトリップ）
- テストで最終的な等価性は `==` (dataclass の auto-generated) で確認

### `proto/storage/scenario_repo.py` 設計方針

```python
import sqlite3
from datetime import datetime, timezone
from pathlib import Path
from dataclasses import dataclass

from proto.engine.models import ImportCondition
from proto.storage.serializer import condition_to_json, condition_from_json


@dataclass
class ScenarioMeta:
    id: int
    name: str
    created_at: str
    updated_at: str


@dataclass
class Scenario:
    id: int
    name: str
    created_at: str
    updated_at: str
    condition: ImportCondition


class ScenarioNotFoundError(Exception):
    pass


class ScenarioNameConflictError(Exception):
    pass


class ScenarioRepository:
    def __init__(self, db_path: Path | str):
        self._db_path = Path(db_path)
        self._db_path.parent.mkdir(parents=True, exist_ok=True)
        self._conn = sqlite3.connect(str(self._db_path))
        self._conn.row_factory = sqlite3.Row
        self._init_schema()

    def _init_schema(self) -> None:
        ...  # CREATE TABLE IF NOT EXISTS ...

    def list_scenarios(self) -> list[ScenarioMeta]: ...
    def get_scenario(self, id: int) -> Scenario: ...
    def save_scenario(self, name: str, cond: ImportCondition) -> int: ...
    # ... 他のメソッド
    def close(self) -> None:
        self._conn.close()
```

**重要**:
- **パラメータバインド必須**: `cursor.execute("... WHERE id = ?", (id,))` でSQLインジェクション防止
- **UNIQUE制約違反**: `sqlite3.IntegrityError` を `ScenarioNameConflictError` に変換
- **現在時刻**: `datetime.now(timezone.utc).isoformat()` で UTC ISO 8601 固定
- **環境変数対応**: デフォルトDBパスは呼び出し側で決定（このクラスは受け取るだけ）

### テスト設計

`pytest` の `tmp_path` fixture を使い、テスト毎に独立したSQLiteファイルを作成:

```python
import pytest
from pathlib import Path
from proto.storage.scenario_repo import ScenarioRepository, ScenarioNameConflictError
from proto.data.mock_params import COND_20FT


@pytest.fixture
def repo(tmp_path: Path) -> ScenarioRepository:
    repo = ScenarioRepository(tmp_path / "scenarios.db")
    yield repo
    repo.close()


def test_save_and_get(repo: ScenarioRepository) -> None:
    sid = repo.save_scenario("標準", COND_20FT)
    scenario = repo.get_scenario(sid)
    assert scenario.name == "標準"
    assert scenario.condition == COND_20FT  # dataclass 等価比較


def test_name_conflict(repo: ScenarioRepository) -> None:
    repo.save_scenario("重複", COND_20FT)
    with pytest.raises(ScenarioNameConflictError):
        repo.save_scenario("重複", COND_20FT)
```
