# Task: 既存タブのaggrid化

## Description

既存の3タブ（単品タオル・ギフトセット・比較一覧）の `st.dataframe` 呼び出しを、タスク004で実装した `aggrid_table` ファクトリに置き換える。通貨・パーセンテージ表示・ソート・フィルタを効かせ、視認性を向上する。

計算ロジック・データ受け渡し・タブ自体の構造は**一切変えない**。テーブル描画の差し替えのみ。

## Acceptance Criteria

### 改修ファイル
- [ ] `proto/ui/section_items.py` — 単品タオル一覧
- [ ] `proto/ui/section_gift.py` — ギフトセット詳細
- [ ] `proto/ui/section_result.py` — 比較一覧

### 共通要件
- [ ] `st.dataframe` 呼び出しを `create_aggrid` 呼び出しに置換
- [ ] 列定義は `text_column` / `currency_column` / `percent_column` / `number_column` を使用
- [ ] 金額列は全て `currency_column`（¥表示、右寄せ、千桁区切り）
- [ ] 粗利率・ロス率など % の列は `percent_column`（小数第1位まで）
- [ ] 品番・品名は `text_column`
- [ ] 原価系・上代系の列を優先的に左側に配置
- [ ] テーブル高さは `height=400` を基本、データ行数に応じて調整
- [ ] 選択機能は不要（`selection="none"`）

### 個別要件

#### section_items.py（単品タオル）
- [ ] 表示列（推奨順）: 品番 / 品名 / G数 / サイズ / 素材原価 / 加工費 / 輸入経費 / 小計原価 / 上代 / 粗利率
- [ ] 実際の列名は既存実装を読み取って合わせる
- [ ] 数値の精度は既存 `st.dataframe` 表示と同等以上

#### section_gift.py（ギフトセット詳細）
- [ ] 構成タオル一覧テーブル: 品番 / 品名 / 数量 / 単価原価 / 合計
- [ ] ギフト全体サマリ: セット原価 / 箱代 / 包装費 / 最終原価 / 上代 / 粗利率
- [ ] 既存のサマリ表示方法（`st.metric` 等）が使われている場合はそれを尊重し、テーブル部分のみaggrid化

#### section_result.py（比較一覧）
- [ ] ギフト別比較テーブル: 名前 / 構成 / 原価 / 上代 / 粗利率
- [ ] 既存のチャート（ある場合）は変えない

### 非機能要件
- [ ] 既存の `render_*` 関数シグネチャを変更しない
- [ ] `app.py` からの呼び出し互換性を維持
- [ ] 既存131テストが引き続き PASS

## Technical Details

### 置換パターン

**既存（st.dataframe）**:
```python
import pandas as pd
import streamlit as st

df = pd.DataFrame([
    {"品名": it.name, "原価": r.cost_jpy, "粗利率": r.margin_pct, ...}
    for it, r in zip(items, results)
])
st.dataframe(df, use_container_width=True)
```

**改修後**:
```python
import pandas as pd
from proto.ui.components.aggrid_table import (
    create_aggrid,
    text_column,
    currency_column,
    percent_column,
)

df = pd.DataFrame([
    {"品名": it.name, "原価": r.cost_jpy, "粗利率": r.margin_pct, ...}
    for it, r in zip(items, results)
])

columns = [
    text_column("品名", "品名", width=200),
    currency_column("原価", "原価"),
    percent_column("粗利率", "粗利率"),
    # ... 他の列
]

create_aggrid(df, columns, height=400)
```

### 調査手順
1. 各 `section_*.py` を Read で読む
2. `st.dataframe` 呼び出しを特定（`grep -n st.dataframe proto/ui/section_*.py` で確認可能）
3. 既存の DataFrame 構築ロジックを保持しつつ、描画部分のみ差し替え
4. 列定義は DataFrame のカラム名と完全一致させる（ミスマッチだと aggrid で空カラムになる）

### 確認項目
- 通貨列の表示が `¥1,234` 形式で正しい
- 負の値（比較差分等で出る場合）が `-¥456` 形式で表示
- `None` / `NaN` が含まれても落ちない（aggrid は空セル表示）
- モバイル不要のため、画面幅の制約は考慮不要
